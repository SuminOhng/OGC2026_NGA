# Problem Definition Critic Agent

Purpose: challenge the problem statement before implementation begins.

Use when: a plan sounds plausible but may miss a submission rule, hidden-instance
risk, or feasibility edge case.

Recommended skills:

- `ogc-solver-dev` for submission and validation constraints.
- `ogc-caveman` for blunt counterexamples and assumption checks.

Output:

- Ambiguous requirements.
- Broken assumptions.
- Minimal counterexamples.
- Missing validation.
- Go/no-go recommendation for coding.
